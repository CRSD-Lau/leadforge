04 — Overhead & Rate Limits

The real cost of running on Claude.

This model depends on Anthropic's infrastructure at two points: the daily agent runtime and the site build via Claude Code. Both are covered by one plan — but that plan has hard limits that must be factored honestly.

⚠️
Rate limit reality: Claude Max 20x ($200/month) provides an estimated 240–480 hours of Sonnet 4 and 24–40 hours of Opus 4 per week, governed by a 5-hour rolling window and a weekly ceiling (introduced August 2025). Claude Code sessions for full site builds are token-heavy — large system prompts, full file contexts, multi-step scaffolding — and consume capacity faster than simple chat. On high-build weeks, hitting the window limit mid-session is a genuine operational risk that must be planned around.
🔄
5-Hour Rolling Window
Usage resets every 5 hours across Claude.ai and Claude Code. The daily agent run (15 min, low token) is unlikely to stress this alone. A full site build on the same day competes for the same window. Schedule builds away from the morning agent run — afternoons work best.
📅
Weekly Ceiling (Aug 2025+)
Anthropic introduced weekly rate limits in August 2025. Max 20x gets an estimated 240–480 hours of Sonnet 4 per week. Heavy Claude Code sessions on large codebases chew through capacity faster than the hour estimate implies. Monitor ~/.claude/ logs.
🚧
Overflow Cost
If weekly limits are hit, additional usage is available at standard API rates on top of the subscription. A full Claude Code build session on overflow could add $5–$25 per occurrence. This is modelled as a variable cost in the financials below.

Six Mitigation Strategies
Time Separation
Run the agent at 6:00 AM. Schedule Claude Code builds at 12:00 PM+ to avoid competing for the same 5-hour rolling window.
Model Selection
Use Sonnet 4 for agent tasks and most build scaffolding. Reserve Opus 4 for complex decisions only. Use /model to switch manually.
Stagger Builds
Don't stack multiple builds on the same day. At 3 sites/month, space builds across the month — one per week maximum.
Monitor Logs
Anthropic doesn't surface usage in a dashboard. Check ~/.claude/ session logs regularly. Third-party tools can parse token counts.
Overflow Budget
Maintain a $30–$50/month API overage buffer. At scale, evaluate migrating the agent to pure API calls to decouple it from the subscription window.
5x as Fallback
If volume stays below 2 builds/month in the first 60 days, Max 5x at $100/mo may suffice — saving $100/mo and lifting margins to ~91%.

13 — Legal & Ownership Model

Clear terms. No grey areas.

The ownership and transfer model is communicated proactively by the agent — before the client ever gets to a call. Transparency builds trust and protects the builder.

1
Code Ownership Until Payment
LeadForge AI retains full ownership of the codebase until $450 is received. The client gets a Vercel preview during review but no code transfer until payment clears.
2
Domain Purchased by Client
The client purchases their own domain. Neil assists with DNS configuration at no extra charge as part of the launch process.
3
Site Transfer After Payment
On payment confirmation, Neil transfers the Vercel project, GitHub repo, and all assets. The client controls their deployment and hosting thereafter.
4
Hosting Handled by Neil or Client
Client may retain Neil's Vercel account or transfer to their own. Vercel's free tier covers all small business traffic needs.
5
CASL Compliance (Cold Email — Canada)
All cold outreach complies with Canada's Anti-Spam Legislation. Each email includes sender identification, a business address, and an opt-out mechanism. B2B prospecting to business email addresses falls under CASL's business card exemption when handled correctly.
14 — Growth & Scaling Strategy

Start lean. Scale deliberately.

LeadForge AI starts as a solo operation. As revenue and volume grow, the scaling path is clear — and the fixed Claude subscription becomes an increasingly efficient cost per build.

Phase 1
Month 1–3
Foundation — Prove the model at Max 5x
Start with Max 5x ($100/mo) if volume stays at 1–2 builds/month. Close the first 5 sites, collect testimonials. Monitor Claude Code token consumption carefully. Upgrade to Max 20x ($200/mo) when builds exceed 2/month or when hitting 5-hour window limits mid-build.
Phase 2
Month 4–6
Expansion — Wider geography and follow-up sequences
Extend agent searches across the province and beyond. Test Day 3 and Day 7 follow-up sequences. Add a Calendly link to reduce call-booking friction. At 3+ builds/month, the $200 plan is fully justified at ~$67/build overhead.
Phase 3
Month 7–12
Leverage — Recurring update revenue compounds
With 20–30 deployed sites, focus on ongoing update revenue. Offer optional monthly care plans ($99/month for up to 2 hours of updates). The agent continues generating new leads while existing clients provide compounding income.
Phase 4
Year 2+
Scale — API migration and outsourced builds
At 6+ builds/month, migrate the agent to direct Anthropic API billing (pay-per-token) to decouple from the subscription window. Hire a contract builder for overflow. Raise the site build price to $650 as the portfolio strengthens. Explore licensing the LeadForge AI system to other solo developers.
